Information Security Principles

🔒 Confidentiality

Information is accessible only to authorized individuals. Access controls and encryption prevent unauthorized disclosure.

✓ Controls: Access lists, encryption, passwords

✓ Risk: Unauthorized access, data leakage

✓ Integrity

Data remains accurate, complete, and unaltered. Information is exactly as it should be.

✓ Controls: Checksums, digital signatures, version control

✓ Risk: Data corruption, unauthorized modification

⚡ Availability

Information is accessible when needed. Systems are operational and responsive to legitimate users.

✓ Controls: Backups, redundancy, load balancing

✓ Risk: Service outages, denial of access

Additional Security Principles

🔑 Authentication

Verifying the identity of users, systems, or devices. "Who are you?"

Methods: Passwords, biometrics, MFA, certificates

🎯 Authorization

Determining what authenticated users can do. "What are you allowed to do?"

Methods: Role-based access, permissions, policies

📝 Non-Repudiation

Users cannot deny their actions. Proof of involvement in transactions.

Methods: Digital signatures, audit trails, logs

📊 Accountability

System tracks who did what and when. Audit trails for all activities.

Methods: Logging, monitoring, forensics

🔍 Key Security Concepts

Vulnerability

A weakness in security that could be exploited.

Example: Unpatched software, weak passwords, misconfigured firewalls

Threat

Any potential attack that could exploit a vulnerability.

Example: Hackers, malware, disgruntled employees, natural disasters

Risk

Probability of a threat exploiting a vulnerability and causing damage.

Formula: Risk = Threat × Vulnerability × Impact

Attack

Active attempt to exploit a vulnerability for malicious purposes.

Example: SQL injection, phishing, DDoS, brute force, ransomware

⚠️ Common Security Threats & Risks

🦠

Malware

Malicious software designed to harm systems or steal data.

Types:

  • Virus - Self-replicating, requires host
  • Worm - Self-replicating, spreads independently
  • Trojan - Disguised as legitimate software
  • Ransomware - Encrypts data, demands payment

⚠️ Impact: Data theft, system damage, financial loss

🎣

Phishing

Social engineering attacks via email or messages to obtain credentials.

Variants:

  • Email phishing - Fake emails from trusted sources
  • Spearphishing - Targeted attacks on specific individuals
  • Whaling - Targets high-level executives
  • Vishing - Voice-based phishing calls

⚠️ Impact: Credential theft, unauthorized access, fraud

💉

SQL Injection

Inserting malicious SQL code to manipulate database queries.

How It Works:

Attacker inputs specially crafted SQL commands in form fields to:

  • Extract sensitive data
  • Modify or delete records
  • Bypass authentication

⚠️ Impact: Data breach, database compromise, system takeover

🌊

DDoS Attack

Overwhelming servers with massive traffic to disable services.

Types:

  • Volume attacks - Consume bandwidth
  • Protocol attacks - Exploit weaknesses
  • Application attacks - Target specific services

⚠️ Impact: Service outages, loss of revenue, reputation damage

🕵️

Man-in-the-Middle (MitM)

Intercepting communication between two parties to steal or modify data.

Methods:

  • ARP spoofing on local networks
  • DNS spoofing - Redirect to fake sites
  • Unencrypted WiFi sniffing

⚠️ Impact: Credential theft, data interception, session hijacking

👤

Insider Threats

Malicious actions by authorized employees or contractors.

Types:

  • Data theft for personal gain
  • Sabotage by disgruntled employees
  • Unintentional security breaches

⚠️ Impact: Severe data loss, operational disruption, legal issues

💣

Zero-Day Vulnerability

Exploitable flaws unknown to vendors, no patches available yet.

Characteristics:

  • No existing fix or patch
  • Highly valuable to attackers
  • Detection is extremely difficult

⚠️ Impact: Rapid system compromise before defense available

📋

Data Breach

Unauthorized access and extraction of sensitive data.

Common Causes:

  • Weak security controls
  • Compromised credentials
  • Unpatched vulnerabilities

⚠️ Impact: Privacy violations, regulatory fines, loss of trust

🎭

Social Engineering

Manipulating people into divulging confidential information or performing actions.

Tactics:

  • Pretexting - Creating false scenarios
  • Baiting - Offering incentives for information
  • Tailgating - Following into restricted areas

⚠️ Impact: Unauthorized access, information disclosure

✍️ Learning Activities

👤 Individual Activity: Security Incident Analysis

Duration: 20 minutes | Individual Work

Scenario: E-Commerce Platform Security Breach

You're a security analyst at an online shopping platform. The company discovered that attackers accessed the database containing 500,000 customer records including names, emails, phone numbers, and encrypted passwords. The attack occurred through an unpatched SQL vulnerability in the login portal that was known for 6 months but not fixed.

✓ Your Tasks:

  1. Identify which CIA Triad principles were violated
  2. Categorize the threats, vulnerabilities, and risks in this scenario
  3. List 5 security controls that could have prevented this breach
  4. Propose immediate actions (short-term) and preventive measures (long-term)
  5. Calculate potential business impact (financial, reputation, legal)

📋 Deliverables:

  • Written analysis (1-2 pages)
  • CIA Triad violation matrix
  • Security recommendations with priority levels

👥 Group Activity: Security Framework Design

Duration: 45 minutes | 4-5 Person Groups

Task: Design an Information Security Framework for a Bank

Your group is hired to design a comprehensive security framework for a regional bank with 500,000 customers. The bank processes daily transactions worth millions and stores highly sensitive financial data.

📌 Deliverables (4 Components):

1. Threat & Vulnerability Analysis

  • Top 6 threats specific to banking
  • Critical vulnerabilities to address
  • Risk prioritization matrix

2. Security Controls & Policies

  • Access control policies (authentication, authorization)
  • Data protection measures (encryption, backup)
  • Incident response procedures

3. Compliance & Governance

  • Regulatory requirements (PCI-DSS, banking regulations)
  • Audit and monitoring requirements
  • Employee training programs

4. Incident Response Plan

  • Detection and response procedures
  • Communication protocols
  • Recovery and lessons learned processes

📊 Presentation Requirements:

  • 10-15 minute presentation with visual slides
  • Clearly explain security framework architecture
  • Justify recommendations with risk analysis
  • Prepare for Q&A about implementation challenges

Five Main Data Collection Methods

📝

Method 1: Manual Data Entry

What it is:

People actively enter data by typing into forms, questionnaires, or surveys.

How it works:

  • User fills out a form with information
  • Data submitted to system (online or on paper)
  • Data entered into database

🔍 Real-World Examples:

🏥 Hospital: Patient fills out intake form with medical history, allergies, current medications
📝 School: Student fills out online registration form with name, address, phone, email
🛍️ Retail: Customer enters credit card info, shipping address at checkout
📋 Survey: Market researcher sends survey asking 50 questions about product preferences

✅ Advantages:

Flexible — can ask complex questions

Detailed — get rich, nuanced responses

Customizable — adapt questions per user

❌ Disadvantages:

Slow — people take time to fill out

Error-prone — typos, wrong values

Low response rate — many skip surveys

Best for: Detailed information, customer preferences, feedback, custom data that varies per person

📱

Method 2: Sensors & IoT Devices

What it is:

Automated devices measure physical conditions and transmit data automatically (no human intervention needed).

How it works:

  • Sensors continuously measure (temperature, motion, location, etc.)
  • Data transmitted automatically to system
  • No manual entry required

🔍 Real-World Examples:

🌡️ Weather Station: Thermometers measure temperature every 5 seconds, data sent to weather service automatically
⌚ Fitness Tracker: Wearable watch counts steps, measures heart rate, records sleep automatically throughout day
🏭 Factory: Sensors on machinery measure temperature, vibration, speed; alert engineers if problems detected
🚗 Smart City: Traffic cameras count vehicles, air quality sensors measure pollution levels

✅ Advantages:

Automatic — no human effort needed

Fast — continuous real-time data

Accurate — machines don't make typos

Volume — can collect millions per second

❌ Disadvantages:

Expensive — hardware costs high

Limited — only measures what sensor measures

Requires maintenance — devices fail, need replacement

Privacy concerns — constant tracking

Best for: Physical measurements, continuous monitoring, real-time alerts, large volumes of consistent data

💳

Method 3: Transaction Data

What it is:

Data automatically captured when customers make purchases, payments, or use services.

How it works:

  • Customer makes transaction (purchase, withdrawal, etc.)
  • System automatically records all details
  • Data includes who, what, when, where, how much

🔍 Real-World Examples:

🛒 E-commerce: Every purchase automatically records: customer ID, items bought, price, date/time, shipping address, payment method
🏦 ATM: Every withdrawal records: account number, amount, location, timestamp, remaining balance
📱 Phone Company: Every call/text automatically logs: from number, to number, duration, timestamp, cost
🎬 Streaming: Every video watched records: user ID, title, duration watched, device, timestamp

✅ Advantages:

Automatic byproduct of business

Complete — every transaction captured

Real behavior — not self-reported

Timestamps — know exactly when

❌ Disadvantages:

Limited to existing customers

Don't know why they bought

Miss non-customers

Privacy concerns — tracking behavior

Best for: Understanding actual customer behavior, revenue tracking, fraud detection, predictive analytics

👁️

Method 4: Observation & Tracking

What it is:

Systematically watching and recording behavior or events without people knowing or being directly involved.

How it works:

  • Researcher observes and takes notes
  • Camera/tracker records movements or events
  • Clicks/interactions tracked digitally
  • Behavior patterns documented

🔍 Real-World Examples:

🛍️ Retail Store: Store cameras track which items customers look at longest, which aisles get traffic, where people spend time
🌐 Website Analytics: Google Analytics tracks which pages users visit, how long they stay, where they click, where they drop off
📍 GPS Tracking: Uber/Lyft track driver location, route taken, speed, time of day
🎓 Education: Learning management system tracks which students accessed materials, how long they studied, when they logged in

✅ Advantages:

Natural behavior — people not influenced

Detailed — capture everything happened

No respondent burden — people don't have to answer

Real patterns — actual behavior

❌ Disadvantages:

Privacy concerns — tracking without consent

Don't know motivation — why they did it

Ethical issues — surveillance concerns

Legal restrictions — GDPR, privacy laws

Best for: Understanding real behavior patterns, UX optimization, fraud detection, location intelligence

🔌

Method 5: APIs & External Data Sources

What it is:

Connecting to external services/databases to access data from other organizations automatically.

How it works:

  • Organization grants permission to access their API
  • Your system requests data programmatically
  • Data flows automatically to your database
  • May happen in real-time or batches

🔍 Real-World Examples:

🗺️ Uber: Uses Google Maps API for location data, traffic patterns, directions
🏦 Banking: Banks use credit reporting agencies' APIs to get credit scores, financial history
📊 Analytics: Companies fetch social media data (via Twitter, Instagram APIs) for sentiment analysis
🌤️ Weather Apps: Pull weather data from national meteorological services via APIs

✅ Advantages:

Don't collect yourself — save time/cost

Always current — real-time data

Authorized — data source gives permission

Reliable — professional sources

❌ Disadvantages:

Dependent on external service availability

May have cost/usage limits

Limited control — can't customize what you get

Service changes API — you must adapt

Best for: Reference data (weather, location), third-party integrations, enriching internal data with external sources

💾 Part 2: Data Storage Techniques

Once you've collected data, you need to store it effectively. Different situations require different storage approaches.

🎯 Storage Decision Factors

Data Structure

Structured (tables) vs Unstructured (images, documents, videos)

Query Speed

Need instant answers (milliseconds) or batch processing (hours) acceptable?

Volume

Gigabytes (small), Terabytes (medium), Petabytes (massive)?

Cost

Budget for hardware, cloud services, maintenance?

📊 1. Relational Databases (SQL)

What it is:

Organized data in tables with rows and columns, linked by relationships. Uses SQL queries to retrieve data.

Popular Systems: MySQL, PostgreSQL, Oracle, Microsoft SQL Server

How it works:

  • Data organized into tables (like Excel spreadsheets)
  • Each row is a record (e.g., one customer)
  • Each column is a field (e.g., customer name, email, phone)
  • Tables linked through keys (relationships)
  • SQL queries retrieve specific data

📋 Example: Customer Database

CustomerID Name Email Phone SignupDate
1001 John Smith john@example.com 555-1234 2024-01-15
1002 Sarah Jones sarah@example.com 555-5678 2024-02-20
1003 Mike Brown mike@example.com 555-9012 2024-03-10

Sample SQL Query:

SELECT Name, Email FROM Customers WHERE SignupDate > '2024-01-01';

This query finds all customers who signed up after January 1st, 2024.

✅ Advantages:

Data integrity — enforces rules, prevents errors
ACID compliant — transactions safe
Relationships — link data across tables
Query flexibility — ask complex questions
Proven — used 50+ years

❌ Disadvantages:

Schema rigid — must define structure upfront
Scaling difficult — hard to handle petabyte scales
Unstructured data — not good for images, videos
Setup complex — requires DBA management

Best For: Business applications (accounting, CRM, HR), structured data, transactional systems, where accuracy critical

🗂️ 2. NoSQL Databases (Non-Relational)

What it is:

Flexible storage for unstructured or semi-structured data. No rigid schema required.

Popular Systems: MongoDB, Cassandra, Firebase, DynamoDB, Redis

Types of NoSQL:

  • Document Stores: Store data as JSON documents (MongoDB)
  • Key-Value Stores: Simple key→value pairs (Redis)
  • Column Stores: Optimized for analytics (HBase)
  • Graph Stores: Store relationships between data (Neo4j)

📋 Example: Document-Based NoSQL

{ "customer_id": "1001", "name": "John Smith", "email": "john@example.com", "orders": [ { "order_id": "ORD-1", "items": ["iPhone 15", "AirPods"], "total": 1200, "date": "2024-01-15" }, { "order_id": "ORD-2", "items": ["MacBook Pro"], "total": 2500, "date": "2024-02-20" } ], "preferences": { "newsletter": true, "language": "English" } }

Note: Different documents can have different fields. Flexible structure!

✅ Advantages:

Flexible schema — add fields anytime
Scalable — handles massive volumes easily
Fast writes — optimized for speed
Unstructured data — great for images, docs, JSON
Cloud-friendly — built for distribution

❌ Disadvantages:

Less strict — data validation not enforced
Complex queries — harder to join data
Duplicates possible — same data stored multiple times
Learning curve — different query language
Consistency issues — eventual consistency

Best For: Real-time apps (chat, gaming), social media, IoT sensor data, mobile apps, massive scale systems

🏗️ 3. Data Warehouses

What it is:

Centralized storage combining data from multiple sources for analysis and reporting (not transactional use).

Popular Systems: Snowflake, BigQuery, Redshift, Azure Synapse

How it works:

  • Data from multiple sources (sales systems, CRM, accounting) loaded
  • Historical data kept (years of history)
  • Optimized for complex analysis queries
  • Separate from production systems (doesn't slow down operations)
  • Used by analysts and executives, not customers

📊 Example: What a Warehouse Contains

Tables in a Sales Warehouse:
  • Customers — all customers ever
  • Products — all products offered
  • Sales — every transaction ever
  • Returns — every return ever
  • Marketing — campaign data
  • Inventory — stock history

Typical Warehouse Queries:

  • "What was total revenue last year by region?"
  • "Which products had highest profit margin?"
  • "How many customers churned vs retained?"
  • "What's the trend in customer spending?"

✅ Advantages:

Centralized — all data in one place
Historical — keep years of data
Analyzed easily — SQL queries get answers
Doesn't slow operations — separate system
Multiple views — organize for different teams

❌ Disadvantages:

High cost — expensive to build/maintain
Complex — many moving parts
Slow setup — takes months to implement
Data quality — only as good as source data
Not real-time — data loaded periodically

Best For: Business intelligence, executive reporting, historical analysis, trend identification, strategic planning

🌊 4. Data Lakes (Cloud Storage)

What it is:

Massive, cheap storage for raw data in any format (structured, unstructured, mixed). Data not organized upfront.

Popular Systems: AWS S3, Azure Data Lake, Google Cloud Storage, Hadoop

How it works:

  • Store everything — photos, videos, logs, files, databases
  • No schema needed — dump data as-is
  • Organize later — process when needed
  • Massive scale — petabytes affordable
  • Process with various tools — SQL, Spark, Python, etc.

📋 Example: What a Data Lake Stores

Everything from daily operations:
  • 📊 Transaction logs (structured)
  • 📸 Customer photos (unstructured)
  • 📹 Security camera feeds (unstructured)
  • 📄 PDF documents (unstructured)
  • 📱 Mobile app events (semi-structured)
  • 🔊 Audio recordings (unstructured)
  • 💬 Chat messages (text)

✅ Advantages:

Cheap — very low storage cost
Scalable — handles unlimited data
Flexible — any format accepted
Fast writes — load data quickly
Flexible analysis — use any tool

❌ Disadvantages:

Messy — "data swamp" if unmanaged
Quality unknown — bad data goes in
Hard to find data — poor organization
Complex querying — not like databases
Security risks — everything exposed

Best For: Machine learning, research, long-term archival, experimental analysis, multi-modal data

📁 5. File Systems & Cloud Storage

What it is:

Simple folder/file storage for documents, media, and backups (like OneDrive, Dropbox, Google Drive).

Popular Systems: Google Drive, OneDrive, Dropbox, AWS S3, iCloud

How it works:

  • Organize files in folders/directories
  • Upload, download, share files easily
  • Version control — keep file history
  • Access control — share with specific people
  • Backup — automatic copies

📊 Typical Organization

📦 My Drive ├── 📁 Projects │ ├── 📁 Website Redesign │ │ ├── 📄 design.psd │ │ ├── 📄 spec.docx │ │ └── 📹 demo.mp4 │ └── 📁 Marketing Campaign │ └── 📊 Q1_Results.xlsx ├── 📁 Reports │ ├── 📄 Annual_Report_2024.pdf │ └── 📊 Monthly_Metrics.xlsx └── 📁 Archive └── 📁 2023

✅ Advantages:

Simple — everyone knows how to use
Accessible — works on any device
Shareable — easy collaboration
Backup — automatic protection
Cheap — affordable cloud plans

❌ Disadvantages:

Not queryable — can't search data inside files
Organization chaos — hard to find files
No relationships — files isolated
Permission management — complex at scale
Doesn't work for operational data

Best For: Document storage, team collaboration, media archival, backups, personal files

🗂️ Part 3: Designing Databases (Schema Design)

Designing a good database is crucial. A poor design leads to errors, slowness, and data corruption. A good design makes everything work smoothly.

📐 Key Design Principles

Organize Related Data

Group similar information in tables (Customers table has customers, Orders table has orders)

Eliminate Redundancy

Don't duplicate data across tables (store customer email once, reference with ID in other tables)

Maintain Data Integrity

Enforce rules (customer must exist before creating order, price must be positive)

Ensure Query Performance

Structure for fast access (indexes on common search fields like CustomerID)

Scenario: Design database for online store. What tables do we need?

Field Type Description Example

Table 4: OrderItems (Which Products in Each Order)

Field Type Description Example
OrderItemID Integer Primary Key 10001
OrderID Integer Foreign Key (links to Orders) 5001
ProductID Integer Foreign Key (links to Products) 2001

UnitPrice Decimal Price at time of order 999.99

How the Relationships Work:

Customer 1001
→
Order 5001
→
Product 2001

Sample Query: "Who bought iPhone 15 and for how much?"

SELECT c.Name, oi.Quantity, oi.UnitPrice FROM Customers c JOIN Orders o ON c.CustomerID = o.CustomerID JOIN OrderItems oi ON o.OrderID = oi.OrderID JOIN Products p ON oi.ProductID = p.ProductID WHERE p.ProductName = 'iPhone 15';

📚 Key Concepts Reference

Primary Key: A unique identifier for each record in a table. No two records can have the same primary key.
Foreign Key: A field that references a primary key in another table, creating relationships between tables.
Schema: The structure of a database — the tables, fields, data types, and relationships defined.
SQL Query: A command to retrieve or manipulate data in a relational database (SELECT, INSERT, UPDATE, DELETE).
Data Warehouse: Centralized storage combining data from multiple sources for analysis and business intelligence.
Data Lake: Massive, flexible storage for raw data in any format (structured, unstructured, or mixed).
ACID Properties: Atomicity, Consistency, Isolation, Durability — ensures database transactions are reliable and safe.
Relational Database: Organizes data in tables with rows and columns, linked through keys (SQL databases).
NoSQL Database: Non-relational database flexible with schema, handles massive scale and unstructured data.
API (Application Programming Interface): Connection allowing systems to automatically share data with each other.

🎓 Key Takeaways

📌

Five Collection Methods: Manual Entry (detailed), Sensors (automatic), Transactions (business activity), Observation (behavior), APIs (external).

📌

Choose the Right Method: Each has pros/cons. Consider speed, accuracy, volume, cost, and what data you need.

📌

Five Storage Types: Relational (business apps), NoSQL (scalable), Warehouse (analytics), Data Lake (raw storage), File Systems (documents).

📌

Good Database Design: Eliminate redundancy, maintain integrity, organize logically, perform efficiently.

📌

Collection + Storage = Success: Right collection methods → quality data. Right storage → accessible, usable data. Both matter!

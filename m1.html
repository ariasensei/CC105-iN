📊 What is a Database?

Definition:

A database is an organized collection of structured data stored electronically in a computer system that can be accessed, managed, and updated efficiently. It allows multiple users to retrieve and modify data simultaneously while maintaining data integrity and security.

🎯 Key Characteristics

  • • Organized: Data structured in tables & relationships
  • • Persistent: Data stored permanently
  • • Shared: Multiple users access simultaneously
  • • Controlled: Access restrictions & permissions
  • • Efficient: Fast retrieval of specific data

📋 Real-World Examples

  • 🏥 Hospital: Patient records, medical history, prescriptions
  • 🏦 Bank: Account info, transactions, customer data
  • 🏪 E-commerce: Products, orders, customer profiles
  • 🎓 University: Student records, grades, course info
  • 📱 Social Media: User profiles, posts, comments

⚙️ What is a DBMS?

Definition:

A Database Management System (DBMS) is software that acts as an intermediary between users and the database. It controls how data is stored, retrieved, updated, and deleted while ensuring data security, consistency, and integrity.

🔑 DBMS Functions

1. Data Definition

Create database structure (tables, fields, relationships)

2. Data Manipulation

Insert, update, delete, and retrieve data

3. Data Retrieval

Query and fetch specific data efficiently

4. Data Security

User authentication, access control, encryption

5. Data Integrity

Ensure data consistency and validity

6. Backup & Recovery

Protect data and recover from failures

🔧 Basic Database Components

📋

1. Tables (Relations)

The basic unit of data storage in a database, organized in rows and columns.

Example: Students Table

StudentID Name Email
101 John Doe john@email.com
102 Jane Smith jane@email.com
  • ✓ Row: Individual record (e.g., one student)
  • ✓ Column: Attribute/field (e.g., name, email)
  • ✓ Cell: Single data value
🔑

2. Primary Key

A unique identifier that uniquely identifies each record in a table.

Example: StudentID = Primary Key

Each StudentID is unique and identifies one student

  • ✓ Must be unique for each record
  • ✓ Cannot be NULL (empty)
  • ✓ Usually auto-incrementing (101, 102, 103...)
  • ✓ Ensures data integrity
🔗

3. Foreign Key

A field that links to the primary key of another table, creating relationships.

Example: Enrollments Table

StudentID in Enrollments = FK (refers to Students table)

  • ✓ References primary key of another table
  • ✓ Maintains referential integrity
  • ✓ Enables table relationships
  • ✓ Prevents invalid data entry
📊

4. Attributes (Columns)

Properties or characteristics of entities stored in each column.

Student Table Attributes:

  • • StudentID (Integer)
  • • Name (Text)
  • • Email (Text)
  • • GPA (Decimal)
  • ✓ Has a data type (Integer, Text, Date, etc.)
  • ✓ May have constraints (NOT NULL, UNIQUE)
  • ✓ Represents single information piece
📝

5. Records (Tuples)

Individual rows containing complete data for one entity.

Example: One Record

{ 101, "John Doe", "john@email.com", 3.8 }

  • ✓ Complete set of related fields
  • ✓ Represents one instance of entity
  • ✓ Can be updated independently
🗂️

6. Schema

The overall structure and organization of the database.

Schema Shows:

  • • All tables in database
  • • Columns in each table
  • • Data types
  • • Relationships between tables
  • ✓ Blueprint of database
  • ✓ Defines constraints & rules
  • ✓ Visualized as ER diagram

📐 Database Component Hierarchy

Database (Instance)

├── Schema (Structure)

│ ├── Table 1

│ │ ├── Column 1 (PK)

│ │ ├── Column 2 (FK)

│ │ └── Column 3

│ │

│ └── Record 1, Record 2, Record 3...

│

└── Table 2

├── Column 1 (PK)

└── Record 1, Record 2...

✍️ Learning Activities

👤 Individual Activity: Database Component Identification

Duration: 20 minutes | Difficulty: Intermediate

📋 Task Instructions:

Given the following scenario, identify and label the database components:

Company Employee Database:

EmpID Name DeptID Salary
1001 Alice Johnson D05 $65,000
1002 Bob Smith D03 $72,000
1003 Carol White D05 $68,500

Questions to Answer:

  1. 1. Identify the Primary Key. Why is it the best choice?
  2. 2. List all attributes (columns) and their data types.
  3. 3. What would be a Foreign Key? Explain.
  4. 4. How many records are shown? Describe one record completely.
  5. 5. What constraints should be applied to each column?

💡 Tip: Think about unique values, relationships to other tables, and data constraints that would ensure data quality.

👥 Group Activity: Design a School Database Schema

Duration: 45 minutes | Group Size: 3-4 students | Difficulty: Advanced

📋 Task Instructions:

Your group must design a database schema for a school system that tracks:

  • ✓ Student information (admission data, contact details)
  • ✓ Teachers and their specializations
  • ✓ Classes/Sections offered
  • ✓ Student enrollment in classes
  • ✓ Grades and academic performance

Your Deliverables:

  1. 1. List all required tables with descriptions
  2. 2. Define columns for each table (with data types)
  3. 3. Identify Primary Keys for each table
  4. 4. Identify Foreign Keys and relationships
  5. 5. Draw an ER Diagram (sketch or digital)
  6. 6. Write sample SQL to create one table

📌 Requirements: Ensure no redundancy, maintain referential integrity, apply appropriate constraints, and create logical relationships.
